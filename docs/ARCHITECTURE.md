## Управление контекстом

### Как бот помнит контекст

1. **Краткосрочная память** — последние N сообщений из БД (по умолчанию 50)
   - Каждое сообщение сохраняется с: chat_id, user_id, username, текст,
     флаг переслано/оригинал, флаг бота/юзера
   - Настраивается через админ-панель

2. **Долгосрочная память** — «ключевые события»
   - Когда сообщений больше N, старые суммаризуются через AI
   - Суммаризация сохраняется как ключевое событие
   - Ключевые события включаются в системный промпт
   - Админ может загружать большой контекст (до 100k символов)

3. **Сборка контекста перед ответом:**
   ```
   Системный промпт:
     ├── Личность (из PERSONALITY.md или БД)
     ├── Тема (если задана админом)
     ├── Кастомные инструкции (если есть)
     ├── Ключевые события чата
     └── Правила поведения
   
   История сообщений:
     ├── [Ключевое событие 1]
     ├── [Ключевое событие 2]
     ├── user: "Привет, Закури!"
     ├── assistant: "Ну привет, дружище!"
     ├── user: "Что думаешь о ...?"
     └── (текущее сообщение)
   ```

### Распознавание пересланных сообщений

Бот определяет пересланные сообщения через Telegram API:
- `message.forward_origin` присутствует → сообщение переслано
- В контексте помечается как `[Переслано от {username}]: {текст}`
- Обычные сообщения: `{username}: {текст}`
- Бот различает оригинальные и пересланные сообщения

## Мульти-провайдерная AI система

```
Запрос к AI
    │
    ▼
Провайдер #1 (priority=0, NVIDIA NIM)
    │ → успех? → возврат результата
    │ → ошибка/лимит? ↓
    │
Провайдер #2 (priority=1, OpenAI)
    │ → успех? → возврат результата
    │ → ошибка/лимит? ↓
    │
Провайдер #3 (priority=2, Groq)
    │ → успех? → возврат результата
    │ → ошибка? ↓
    │
Все провайдеры упали → ошибка
```

- Текстовые и image-провайдеры управляются отдельно
- Каждый провайдер: name, base_url, api_key, model, priority, is_active
- OpenAI-совместимый формат API (подходит для большинства провайдеров)
- Управление через админ-меню в Telegram

## Генерация фото

### Генерация нового фото
1. Пользователь пишет: «Закури, нарисуй [промпт]»
2. Бот извлекает промпт (обязателен)
3. Если промпта нет → бот просит уточнить
4. Вызов image API провайдера
5. Отправка фото в чат

### Редактирование фото
1. Пользователь reply на фото: «Закури, измени [инструкции]»
2. Бот скачивает оригинальное фото
3. Вызов image edit API с фото + инструкциями
4. Отправка результата в чат

### Ключевые слова
- Генерация: «нарисуй», «сгенерируй», «создай фото», «сделай картинку»
- Редактирование: «измени», «отредактируй», «переделай», «модифицируй»
