# План разработки Telegram-бота «Дракончик Закури»

## Обзор

Плюшевый дракон-бот для Telegram-чатов. Общается через AI, помнит контекст,
генерирует и редактирует фото, настраивается через меню в личке.

## Фазы разработки

### Фаза 1: Базовая структура ✅
- `main.py` — точка входа, деплой на Railway
- Подключение к Telegram через aiogram 3.x
- SQLite база данных (aiosqlite)
- Базовые обработчики сообщений

### Фаза 2: AI интеграция ✅
- Мульти-провайдерная система (OpenAI-совместимые API)
- Fallback: если один провайдер упал/лимит — следующий
- Поддержка NVIDIA NIM, OpenAI, Groq, Together, и любых OpenAI-совместимых
- Добавление/удаление провайдеров через админ-меню

### Фаза 3: Личность и память ✅
- Личность загружается из `docs/PERSONALITY.md`
- Хранение контекста чата в БД (последние N сообщений)
- Суммаризация старых сообщений в «ключевые события»
- Загрузка до 100k символов контекста через личку
- Распознавание пересланных сообщений

### Фаза 4: Админ-панель ✅
- Inline-меню в личке (кнопки, не команды)
- Управление AI-провайдерами (добавить/удалить/включить/выключить)
- Настройка базового промпта и темы
- Управление контекстом и памятью
- Добавление/удаление админов

### Фаза 5: Генерация фото ✅
- Обнаружение запросов на генерацию (ключевые слова + промпт)
- Генерация через image API провайдеров
- Редактирование существующих фото (reply на фото + инструкции)
- Обязательное наличие промпта от пользователя

### Фаза 6: Поведение в чате ✅
- Отвечает при упоминании (@бот, «закури»)
- Отвечает при reply на его сообщение
- Случайные авто-ответы (настраиваемая частота)
- Грубые ответы на оскорбления (через personality промпт)
- Нет цензуры, кроме политики

## Технологии

| Компонент | Технология |
|-----------|-----------|
| Язык | Python 3.11+ |
| Telegram | aiogram 3.x |
| HTTP | aiohttp |
| База данных | aiosqlite (SQLite) |
| Деплой | Railway |
| AI API | OpenAI-совместимый формат |

## Деплой на Railway

1. Загрузить код на GitHub
2. Подключить Railway к репозиторию
3. Установить переменные окружения (см. `.env.example`)
4. Railway автоматически запустит `python main.py`
5. Бот работает через long-polling (не нужен webhook)

## Переменные окружения

| Переменная | Обязательно | Описание |
|-----------|-------------|----------|
| `BOT_TOKEN` | Да | Токен от @BotFather |
| `ADMIN_IDS` | Да | ID админов через запятую |
| `DB_PATH` | Нет | Путь к БД (по умолчанию `bot.db`) |
| `DEFAULT_PROVIDER_URL` | Нет | URL AI-провайдера по умолчанию |
| `DEFAULT_PROVIDER_KEY` | Нет | API ключ провайдера по умолчанию |
| `DEFAULT_PROVIDER_MODEL` | Нет | Модель провайдера по умолчанию |

## Управление

Все настройки доступны через `/admin` в личке бота:
- Добавление/удаление AI-провайдеров (text и image)
- Настройка промпта, темы, инструкций
- Управление контекстом и ключевыми событиями
- Настройка частоты ответов
- Просмотр статистики
