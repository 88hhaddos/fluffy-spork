## Фаза 1: Модуль Football API (`src/football_api.py`)

### Эндпоинты которые используем:
```
GET /games/list?today=true          — матчи сегодня
GET /games/list?upcoming=true       — предстоящие матчи
GET /games/list?live=true           — live матчи
GET /games/list?league=X&year=Y     — матчи лиги за сезон
GET /games/{id}                     — полные данные матча
GET /games/glicko/{id}              — прогноз вероятностей
GET /games/last-games-stats?gameId  — форма команд перед матчем
GET /games/text-summary?id          — текстовая сводка (коэф, xG, прибыльность)
GET /games/profits?gameId           — прибыльность ставок
GET /games/injuries?gameId          — травмы/дисквалификации
GET /games/season-table?league=X    — турнирная таблица
GET /leagues                        — список лиг (для поиска ID лиг ЧМ)
```

### Функции модуля:
- `get_today_matches()` — матчи на сегодня (команды, время, коэффициенты)
- `get_upcoming_matches(limit=10)` — ближайшие матчи
- `get_match_details(game_id)` — полные данные (статистика, составы, события)
- `get_match_prediction(game_id)` — прогноз Glicko 2 (вероятности %)
- `get_team_form(game_id)` — форма команд (последние N матчей)
- `get_match_summary(game_id)` — текстовая сводка для промпта
- `get_standings(league_id, year)` — турнирная таблица
- `get_injuries(game_id)` — травмы
- `find_league(name)` — поиск ID лиги по названию (ЧМ, АПЛ, etc)
- `get_live_matches()` — live матчи (результаты в реальном времени)

### Кэширование:
- Матчи сегодня — кэш 15 минут
- Данные матча — кэш 5 минут (live) / 1 час (завершён)
- Таблица — кэш 1 час
- Прогноз — кэш 1 час

## Фаза 2: Модуль ставок (`src/betting.py`)

### БД — таблица `bot_bets`:
```sql
CREATE TABLE bot_bets (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    chat_id INTEGER NOT NULL,
    bet_type TEXT,          -- 'single' | 'express'
    match_ids TEXT,         -- ID матчей через запятую
    selections TEXT,        -- что поставил (JSON)
    odds REAL,              -- коэффициент
    stake REAL,             -- сумма ставки (виртуальная)
    status TEXT,            -- 'pending' | 'won' | 'lost' | 'void'
    potential_return REAL,  -- возможный выигрыш
    actual_return REAL,     -- реальный выигрыш
    created_at TIMESTAMP,
    settled_at TIMESTAMP
);

CREATE TABLE bot_balance (
    chat_id INTEGER PRIMARY KEY,
    balance REAL DEFAULT 10000,  -- виртуальный баланс
    total_won REAL DEFAULT 0,
    total_lost REAL DEFAULT 0,
    bets_count INTEGER DEFAULT 0
);
```

### Логика бота-лудомана:
- Стартовый баланс: 10 000 виртуальных единиц
- Размер ставки: 500-2000 (рандомно, но не больше 20% баланса)
- Типы ставок:
  - **Одиночная (single)**: победа / ничья / тотал больше/меньше / обе забьют
  - **Экспресс (express)**: 2-3 матча, коэффициенты перемножаются
- Бот запоминает все свои ставки
- После матча — проверяет результат, считает выигрыш/проигрыш
- Ведёт статистику: баланс, ROI, процент выигрышей

## Фаза 3: Промпт лудомана

### Обновление личности в `personality.py`:
```
## Футбольный эксперт и лудоман
Ты — яростный фанат Аргентины. Месси — твой кумир.
Ты обожаешь ставить на футбол. Ты лудоман и гордишься этим.
Когда обсуждают матчи — ты споришь, отстаиваешь позицию уверенно.
Даёшь прогнозы с уверенностью эксперта, даже если не уверен.
Используй реальные данные которые тебе предоставляют.
Ты помнишь свои ставки и их результаты.
Хвастаешься выигрышами, злишься при проигрышах.
Аргентина — лучшая сборная мира. Кто не согласен — тот не разбирается в футболе.
```

### При вопросе о матче:
1. Бот определяет что вопрос про футбол (матч, команда, ставка, прогноз)
2. Запрашивает данные из SStats API
3. Формирует контекст с реальными данными
4. AI генерирует ответ с прогнозом и уверенностью

## Фаза 4: Команды и интеграция

### Команды:
```
/matches          — матчи сегодня
/matches tomorrow — матчи завтра
/live             — live матчи (результаты сейчас)
/predict          — прогноз на ближайший матч (reply на матч)
/table            — турнирная таблица (нужно указать лигу)
/table wc         — таблица ЧМ
/bets             — ставки Закури (история + баланс)
/balance          — баланс Закури
```

### Интеграция в чат:
- Юзер: «закури кто победит Аргентина или Франция»
  → бот ищет матч в API → берёт прогноз Glicko 2 → отвечает с данными
- Юзер: «закури какая форма у Аргентины»
  → бот берёт last-games-stats → отвечает со статистикой
- Юзер: «закури коэффициенты на матч»
  → бот берёт данные матча → отвечает с коэффициентами
- Юзер: «закури кто лучший бомбардир ЧМ»
  → бот гуглит / берёт из памяти

### Авто-постинг ставок:
- Background task: каждые 30-120 минут (рандомно)
- Берёт матчи сегодня/завтра
- Выбирает ставку (single или express)
- Сохраняет в БД
- Постит в чат
- После матча — проверяет результат, постит исход
